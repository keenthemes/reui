title: Registry
description: Learn how to use the ReUI Registry with shadcn/ui.
---

The ReUI Registry uses a single `@reui` namespace. The shadcn CLI resolves the full alias suffix into the registry URL, so you can install everything from one namespace.

## Setup ReUI Registry

Add the ReUI registry namespace to your `components.json`. Learn more about registry config from [shadcn registry docs](https://ui.shadcn.com/docs/registry).

```json
{
  "registries": {
    "@reui": "https://reui.io/r/{style}/{name}.json"
  }
}
```

## Installing Components

Component examples use the `c-*` naming pattern and install with no license key or account:

```bash
npx shadcn@latest add @reui/c-alert-1
```

Some `c-*` items pull shared `@reui/*` primitives such as `@reui/alert` or `@reui/data-grid` as registry dependencies. Those supporting files are resolved automatically, so installs keep working out of the box.

## Styles

The registry ships every component in multiple styles. Set your preferred style in `components.json` (for example `base-nova` or `radix-nova`) and the CLI pulls the matching source.
